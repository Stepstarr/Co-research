# 汽车评论消费者需求挖掘研究

> 基于 LLM-ELO 框架的方法创新

---

## 研究问题

**核心问题**：如何从稀疏、短小的汽车评论中挖掘有决策价值的消费者需求？

**现有方法的局限**：
- 筛选 + 简单聚类 → 太笼统
- 缺乏细粒度分析
- 无法支撑具体决策

---

## 创新方向

### 1. 多维度需求图谱

**思路**：不只分类，构建需求之间的关系网络

**维度**：
- 需求类别（油耗、空间、动力、外观、内饰、配置等）
- 情感强度（强烈正向/弱正向/中性/弱负向/强烈负向）
- 需求重要性（基于提及频率和情感强度）
- 需求紧迫性（问题严重程度）
- 依赖关系（哪些需求相互关联）

### 2. 细粒度属性提取

**思路**：从短文本中精确提取具体属性

**方法**：
- LLM 进行属性-情感联合抽取
- 归一化属性名称（"油耗" = "耗油" = "油耗高"）
- 提取情感原因

### 3. 时序演化分析

**思路**：追踪需求的变化趋势

**分析**：
- 新兴需求（近期出现增多）
- 衰退需求（近期出现减少）
- 周期性需求（节假日/新车发布季）
- 突发问题（负面情感突然增加）

### 4. 对比学习

**思路**：通过对比发现差异化需求

**对比维度**：
- 品牌对比
- 车系对比
- 价格区间对比
- 时间段对比

### 5. 决策支撑输出

**最终目标**：可执行的决策建议

**输出格式**：
```
{
  "关键需求": [
    {
      "需求": "油耗",
      "重要性": 0.85,
      "情感分布": {"正向": 60%, "中性": 20%, "负向": 20%},
      "趋势": "下降",  // 负面评价在增加
      "建议": "重点优化油耗表现"
    }
  ]
}
```
