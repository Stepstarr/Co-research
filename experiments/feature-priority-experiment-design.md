# 产品功能优先级排序实验设计

> 基于 LLM-ELO 框架的扩展应用

---

## 研究问题

**核心问题**: 如何利用 LLM-ELO 框架自动化产品功能优先级排序？

**子问题**:
1. 如何定义"需求价值"的评估标准？
2. 如何量化 Impact、Effort、Strategic Value？
3. 成对比较时如何权衡多个维度？

---

## 方法设计

### 第一阶段：LLM 评分

**输入**: 产品需求列表（描述、用户反馈、业务背景）

**评估维度**:
1. **Impact** (影响力): 
   - 影响用户数量
   - 解决问题的重要性
   - 业务价值

2. **Effort** (开发成本):
   - 技术复杂度
   - 所需资源
   - 时间成本

3. **Strategic Value** (战略价值):
   - 与产品愿景的一致性
   - 差异化优势
   - 长期价值

**输出**: 每个需求的维度得分 + 加权总分 + 筛选后的高价值需求集

### 第二阶段：ELO 成对比较

**方法**: 
- 对 Top 20% 的需求进行成对比较
- LLM 作为"产品专家"判断哪个需求优先级更高
- 使用概率归一化的 ELO 更新

**Prompt 设计**:

```
你是一位资深产品经理。请比较以下两个需求，判断哪个应该优先开发。

需求 A: [描述]
- Impact 得分: X/10
- Effort 得分: X/10  
- Strategic Value 得分: X/10

需求 B: [描述]
- Impact 得分: X/10
- Effort 得分: X/10
- Strategic Value 得分: X/10

请综合考虑：
1. 哪个需求对用户价值更大？
2. 哪个需求开发成本更低？
3. 哪个需求更符合产品战略？

输出格式:
{
  "winner": "A/B/Tie",
  "confidence": "High/Medium/Low",
  "rationale": "详细理由..."
}
```

**输出**: 最终优先级排序 + 每个决策的理由

## 数据需求

### 理想数据集

1. **产品需求数据集**:
   - 需求描述
   - 用户反馈数量
   - 实际开发优先级 (作为 ground truth)
   - 开发后的业务指标 (可选)

2. **公开数据集选项**:
   - GitHub Issues (开源项目的功能请求)
   - Product Hunt 评论和需求
   - UserVoice / Canny 等反馈平台数据

### 最小可用数据集

如果无法获取真实产品数据，可以：
1. 构建模拟数据集（10-20 个典型需求）
2. 邀请产品经理进行人工评估作为 baseline
3. 验证框架可行性

---

## 评估指标

1. **与人工排序的一致性**: Spearman 相关系数
2. **Top-K 准确率**: 前 K 个需求的命中准确率
3. **稳定性**: 多次运行的排序一致性
4. **可解释性**: 决策理由的质量评估
